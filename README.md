# 🧠 SENN — Self-Expanding Neural Network
### *"Toward an Artificial Child-Like Learning System"*

A neural network that **grows its own architecture**, **remembers past knowledge**, 
and **commits its own evolution to GitHub** — like a digital child learning over time.

---

## 📁 Project Structure

```
senn/
├── dynamic_net.py          ← Core: DynamicLayer + SelfExpandingNet
├── growth_controller.py    ← When/Where/How to grow (NES-based)
├── memory_manager.py       ← EWC: prevents catastrophic forgetting
├── github_self_modifier.py ← Self-modifies code + commits to git
├── train.py                ← Master training loop
├── dynamic_net_evolved.py  ← AUTO-GENERATED by the network (do not edit)
├── growth_journal.json     ← AUTO-GENERATED growth log
└── training_log.json       ← AUTO-GENERATED training metrics
```

## 🧬 How It Grows

```
Training Step
     ↓
Compute NES = ||grad||² / (||weight||² + ε)   ← Mitchell et al.
     ↓
NES > τ ?   → ADD neurons to that layer       ← WIDTH growth
     ↓
Loss plateau? → ADD neurons to smallest layer  ← DEN paper
     ↓
Output entropy > 80%? → ADD new layer         ← DEPTH growth
     ↓
Weak activations? → PRUNE dead neurons        ← DEN group-sparsity
     ↓
Rewrite dynamic_net_evolved.py                ← Self-modification
     ↓
git commit "Gen N: added 4 neurons because…" ← GitHub self-commit
```

## ⚠️ Honest Limitations

1. **NES is approximated** — true Fisher matrix is too expensive; we use diagonal approx
2. **Web learning is noisy** — filtering needed before training on raw web text
3. **Self-modifying code** rewrites architecture description, not the training logic itself
4. **Memory grows** with tasks — prune Fisher matrices for old tasks if memory-constrained
